## ADDED Requirements

### Requirement: Skill Loader reads Agent Layer and generates IDE context files
The Skill Loader SHALL be a Life Console Skill that reads all Agent Layer files (axioms/, methodologies/, skills/) and generates formatted context Markdown files for configured target paths.

#### Scenario: Skill Loader generates Claude Code context file
- **WHEN** Skill Loader is executed with target "claude-code"
- **THEN** a formatted Markdown file is written to `~/.claude/CLAUDE.md` (appended under a `# Life Console Context` section, not overwriting existing content)

#### Scenario: Skill Loader generates Cursor context file
- **WHEN** Skill Loader is executed with target "cursor"
- **THEN** a file is written to `.cursor/rules/life-console.md` in the current working directory, or to a globally configured path

### Requirement: Generated context file format
The generated context file SHALL contain: a header with generation timestamp and MCP Server URL, all axioms formatted as a numbered list, relevant methodologies (all by default), MCP tool usage instructions, and a note directing the AI to query `searchKnowledge()` for detailed knowledge retrieval.

#### Scenario: Generated file includes generation metadata
- **WHEN** Skill Loader writes a context file
- **THEN** the file begins with `<!-- Generated by Life Console Skill Loader at <timestamp> -->` and includes the MCP server URL

#### Scenario: Generated file is under token budget
- **WHEN** Skill Loader generates a context file for a user with 20 axioms and 5 methodologies
- **THEN** the generated file is under 4000 tokens

### Requirement: Skill Loader auto-triggered after distillation
The system SHALL automatically trigger Skill Loader after a successful distillation-chat write-back to the Agent Layer, for all previously configured targets.

#### Scenario: Auto-trigger after distillation
- **WHEN** distillation-chat writes a new axiom to the Agent Layer
- **THEN** Skill Loader is automatically executed for all configured targets within 5 seconds

### Requirement: Skill Loader configurable targets stored in settings
The system SHALL store Skill Loader target configurations (target name, output path, enabled/disabled) in Life Console settings, persisted to the settings table.

#### Scenario: User adds a new target
- **WHEN** user adds a target with name "cursor" and path "/Users/user/.cursor/rules/life-console.md"
- **THEN** the target is saved and appears in future Skill Loader executions

### Requirement: Skill Loader records run in Context Store
Each Skill Loader execution SHALL create a record in the `records` table with `skill_name="skill-loader"`, the list of targets written as input, and the list of successfully written file paths as output.

#### Scenario: Successful run recorded
- **WHEN** Skill Loader writes to 2 targets successfully
- **THEN** a record is inserted with both target paths in the output JSON
